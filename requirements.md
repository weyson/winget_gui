# Winget GUI 软件更新管理器需求文档

## 项目概述

开发一个基于 Windows 系统的图形界面应用程序，用于管理 Windows 软件的更新。该应用程序通过调用 Windows 包管理器（winget）的 `upgrade` 功能，为用户提供友好的界面来查看和管理需要更新的软件。

## 核心需求

### 1. 图形用户界面 (GUI)

**需求描述：**
- 提供直观、易用的图形界面
- 界面应简洁明了，适合普通用户操作
- 更新过程中界面保持响应，不冻结
- 界面包含日志区域，显示更新进度和详细信息

**技术要求：**
- 支持 Windows 操作系统
- 界面响应流畅，使用多线程处理耗时操作
- 日志区域位于主窗口底部，支持滚动浏览
- 使用 tkinter Text 控件显示实时日志

### 2. 软件列表显示

**需求描述：**
- 显示所有可更新的 Windows 软件列表
- 每个软件项应显示以下信息：
  - 复选框（用于选择）
  - 软件名称
  - 软件 ID
  - 当前版本
  - 可用版本（最新版本）
  - 软件来源（如：winget, msstore 等）

**功能要求：**
- 列表应以表格形式清晰展示
- 支持滚动浏览
- 软件信息解析准确，正确处理中英文混合名称和显示宽度

### 3. 刷新功能

**需求描述：**
- 提供"刷新"按钮，用于获取最新的可更新软件列表
- 点击刷新后，调用 `winget upgrade` 命令重新扫描系统软件

**功能要求：**
- 刷新时显示加载状态
- 刷新完成后自动更新软件列表
- 提供刷新状态的视觉反馈（如：进度条、加载动画）
- 后台线程执行刷新，界面保持响应

### 4. 选择功能

#### 4.1 全选功能
**需求描述：**
- 提供"全选"复选框
- 点击全选后，列表中所有可更新的软件项都被选中

**功能要求：**
- 全选复选框应位于工具栏
- 全选状态与单选状态同步（当所有单项被选中时，全选自动勾选；反之亦然）

#### 4.2 取消全选功能
**需求描述：**
- 提供"取消全选"按钮
- 点击后取消所有软件的选择

**功能要求：**
- 取消全选按钮位于"全选"复选框旁边
- 点击后全选复选框自动取消勾选

#### 4.3 单选功能
**需求描述：**
- 每个软件项第一列有独立的复选框
- 用户可以单独选择或取消选择某个软件

**功能要求：**
- 点击任意列都可以切换复选框状态
- 单选状态实时更新
- 选中行高亮显示（背景色）
- 状态栏实时显示选中数量（如：已选择 3/6 项）

### 5. 更新执行功能

**需求描述：**
- 提供"更新所选"按钮
- 执行更新时调用 `winget upgrade <package-id>` 命令

**功能要求：**
- 仅对选中的软件执行更新
- 更新前显示确认对话框
- 显示更新进度（如：正在更新 XX (1/6)...）
- 更新过程中界面保持响应（后台线程执行）
- 实时获取并显示更新进度信息：
  - 在界面底部显示滚动日志区域
  - 每个软件的更新进度独立显示，格式为 `[软件名称] 进度信息`
  - 显示下载进度、安装状态等详细信息
  - 使用 `subprocess.Popen` 逐行读取 winget 输出
  - 通过回调函数实时更新日志区域
- 更新完成后显示汇总结果（成功/失败统计和详细列表）
- 更新完成后自动恢复按钮可操作状态
- 更新完成后自动刷新软件列表
- 支持批量更新多个软件

## 技术要求

### 命令行交互
- 应用程序需要在后台调用 `winget` 命令
- 需要处理的命令：
  - `winget upgrade` - 获取可更新软件列表
  - `winget upgrade <package-id>` - 更新指定软件
- 使用 `subprocess.Popen` 逐行读取输出，支持实时进度显示
- 使用进度回调函数将输出传递到 GUI 界面

### 数据解析
- 解析 winget 命令的输出，提取软件信息
- 使用从右往左的正则表达式解析，提高兼容性
- 正确处理中英文混合名称（考虑显示宽度差异）

### 权限要求
- 某些软件更新可能需要管理员权限
- 应用程序应能处理权限提升请求

### 多线程处理
- 刷新和更新操作在后台线程执行
- 使用 `root.after` 在主线程更新 UI

## 用户交互流程

1. 用户启动应用程序
2. 应用程序自动刷新并显示可更新软件列表
3. 用户查看列表，选择需要更新的软件（可全选、取消全选或单选）
4. 用户点击"更新所选"按钮
5. 显示确认对话框
6. 后台线程执行更新，界面显示进度
7. 更新完成后显示汇总结果
8. 自动刷新软件列表

## 非功能性需求

### 性能要求
- 列表加载时间：< 3秒
- 界面响应时间：< 500毫秒
- 更新过程中界面保持响应

### 可用性要求
- 界面简洁，无需复杂学习即可使用
- 提供清晰的操作提示和错误信息
- 状态栏显示选中数量

### 兼容性要求
- 支持 Windows 10 及以上版本
- 需要系统已安装 winget（Windows Package Manager）

### 6. 多语言支持

**需求描述：**
- 支持6种语言：简体中文、繁体中文、英语、俄语、日语、韩语
- 首次启动时自动检测系统默认语言并应用
- 用户可手动切换语言，切换后即时生效，无需重启
- 持久化存储用户语言偏好设置
- 后续启动时优先加载用户保存的语言设置

**功能要求：**
- 所有界面元素（按钮、标签、对话框等）支持多语言
- 所有提示信息、错误消息支持多语言
- 在工具栏提供语言选择下拉框，语言标签显示在下拉框左侧
- 语言标签本身也支持多语言显示（根据当前语言动态切换）
- 使用 JSON 文件存储翻译内容
- 语言切换实时刷新所有界面文本

**技术要求：**
- 使用国际化类管理翻译加载和语言切换
- 翻译文件存储在 `locales/` 目录下
- 支持占位符替换（如 `{count}`, `{name}`）
- 用户语言设置存储在 `config.json` 文件中
